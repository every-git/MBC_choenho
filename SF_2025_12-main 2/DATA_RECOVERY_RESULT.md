   - 위치: `~/.dbclient/storage/1765953041431@@127.0.0.1@3306@springdb/history.sql`
   - 내용: 테이블 생성 및 VIEW 생성 쿼리만 있음
   - 실제 데이터 INSERT 문 없음

2. **쿼리 파일**
   - 위치: `~/.dbclient/query/1765953041431@@127.0.0.1@3306@springdb/New_Query_1765956590081.sql`
   - 내용: 테스트 데이터 3개 (bno: 49899, 49898, 49897)
   - 실제 프로덕션 데이터는 아님

3. **MySQLWorkbench autosave 파일**
   - 위치: `~/Library/Application Support/MySQL/Workbench/sql_workspaces/`
   - 내용: 비어있음

### ❌ 찾지 못한 것들

1. **SQL 덤프 파일** (백업 파일)
   - 홈 디렉토리: 없음
   - 외부 드라이브: 없음
   - 프로젝트 디렉토리: 없음

2. **Time Machine 백업**
   - 접근 권한 문제로 직접 확인 불가
   - 수동으로 확인 필요

3. **실제 프로덕션 데이터**
   - SQL 히스토리에 INSERT 문 없음
   - 데이터베이스 파일에서도 복구 불가

## 복원 가능한 데이터

### 테스트 데이터 (선택적 복원)

`New_Query_1765956590081.sql` 파일에 있는 테스트 데이터를 복원할 수 있습니다:

```sql
-- 테스트 게시글 3개
INSERT INTO tbl_board (bno, title, content, writer) VALUES 
(49899, '테스트 게시글 49899', '이 게시글에는 댓글이 200개 있습니다.', '테스트작성자'),
(49898, '테스트 게시글 49898', '이 게시글에는 댓글이 200개 있습니다.', '테스트작성자'),
(49897, '테스트 게시글 49897', '이 게시글에는 댓글이 200개 있습니다.', '테스트작성자')
ON DUPLICATE KEY UPDATE title=title;
```

## 결론

**실제 프로덕션 데이터는 복원할 수 없습니다.**

이유:
1. SQL 덤프 파일이 없음
2. Time Machine 백업 접근 불가 (수동 확인 필요)
3. SQL 히스토리에 실제 데이터 INSERT 문 없음
4. 데이터베이스 파일에서 복구 불가

**권장 사항:**
1. Time Machine을 수동으로 확인 (시스템 설정 → Time Machine)
2. 다른 백업 위치 확인 (클라우드, 외부 드라이브 등)
3. 데이터가 없다면 새로 입력
