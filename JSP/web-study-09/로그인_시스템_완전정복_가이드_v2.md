## 1. 시스템 전체 구조 이해하기

### 🏗️ 전체 그림 (비유: 고급 레스토랑 시스템)

```
[고객] → [웨이터] → [주방장] → [냉장고(DB)] → [VIP룸(세션)]
  ↓         ↓          ↓            ↓            ↓
브라우저  Servlet   DAO        Oracle DB    HttpSession
```

**비유 설명:**
- **고객(브라우저)**: 로그인을 원하는 사용자
- **웨이터(Servlet)**: 주문을 받고 결과를 전달하는 중간 역할
- **주방장(DAO)**: 실제로 냉장고(DB)에서 재료를 찾아오는 역할
- **냉장고(DB)**: 회원 정보가 저장된 곳
- **VIP룸(세션)**: 로그인한 고객을 위한 특별한 공간

### 📁 파일 구조 (업데이트됨)

```
web-study-09/
├── src/main/
│   ├── java/com/saeyan/
│   │   ├── controller/
│   │   │   └── LoginServlet.java    ← 웨이터 (요청 처리) ✨개선됨
│   │   └── dto/
│   │       ├── MemberDAO.java       ← 주방장 (DB 작업) ✨개선됨
│   │       └── MemberVO.java        ← 그릇 (데이터 담는 용기)
│   └── webapp/
│       ├── member/
│       │   ├── login.jsp            ← 메뉴판 (로그인 화면) ✨개선됨
│       │   └── main.jsp             ← 🆕 VIP룸 (로그인 후 화면)
│       ├── css/
│       │   ├── login.css            ← 🆕 로그인 화면 스타일
│       │   └── member-main.css      ← 🆕 메인 화면 스타일
│       └── WEB-INF/
│           └── web.xml              ← 식당 규칙서
```

### ☕ 2. LoginServlet.java - "전문 웨이터" (요청 처리)

**역할**: 사용자의 요청을 받아서 처리하고 결과를 전달

**🆕 새로운 기능:**
- 로그인 성공 시 완전한 회원 정보 세션 저장
- 실패 시 아이디 값 유지 기능
- 올바른 경로로 포워딩

#### ① GET 요청 처리 (doGet) - "메뉴판 보여주기"

```java
protected void doGet(HttpServletRequest request, HttpServletResponse response) 
        throws ServletException, IOException {
    request.getRequestDispatcher("member/login.jsp")
        .forward(request, response);
}
```

#### ② POST 요청 처리 (doPost) - "🆕 향상된 주문 처리"

```java
protected void doPost(HttpServletRequest request, HttpServletResponse response) 
        throws ServletException, IOException {
    request.setCharacterEncoding("UTF-8");
    
    // 1. 폼 데이터 받기
    String userid = request.getParameter("userid");
    String pwd = request.getParameter("pwd");
    String url = "/member/login.jsp";  // 기본 경로 (절대 경로)
    
    // 2. DAO를 통한 로그인 검증
    MemberDAO dao = MemberDAO.getInstance();
    int result = dao.userCheck(userid, pwd);
    
    if(result == 1) {
        // 🆕 로그인 성공 - 완전한 회원 정보 저장
        HttpSession session = request.getSession();
        session.setAttribute("userid", userid);
        
        // 🆕 전체 회원 정보 조회 및 세션 저장
        MemberVO mvo = dao.getMember(userid);
        session.setAttribute("loginUser", mvo);
        
        url = "/member/main.jsp";  // 🆕 메인 페이지로 이동
        
    } else if(result == 0) {
        // 🆕 로그인 실패 - 아이디 값 유지
        request.setAttribute("message", "아이디 또는 비밀번호가 맞지 않습니다.");
        request.setAttribute("userid", userid);  // 🆕 아이디 값 유지
        
    } else {
        // 🆕 DB 오류 - 아이디 값 유지
        request.setAttribute("message", "데이터베이스 연결 오류가 발생했습니다.");
        request.setAttribute("userid", userid);  // 🆕 아이디 값 유지
    }
    
    // 🆕 RequestDispatcher를 통한 포워딩
    RequestDispatcher dis = request.getRequestDispatcher(url);
    dis.forward(request, response);
}
```

**🔧 주요 개선 사항:**
1. **완전한 회원 정보 저장**: `loginUser` 세션에 `MemberVO` 객체 저장
2. **실패 시 아이디 유지**: `request.setAttribute("userid", userid)`
3. **절대 경로 사용**: `/member/login.jsp`, `/member/main.jsp`
4. **올바른 포워딩**: `RequestDispatcher` 사용

### 🍳 3. MemberDAO.java - "마스터 셰프" (DB 작업)

**역할**: 데이터베이스와 직접 소통하는 전문가

**🆕 새로운 기능:**
- `getMember()` 메서드 NPE 방지
- 완전한 회원 정보 조회 기능

#### ① 🆕 개선된 getMember() 메서드

```java
public MemberVO getMember(String userid) {
    MemberVO mvo = null;  // 초기값 null
    String sql = "select * from member where userid = ?";
    
    Connection con = null;
    PreparedStatement pstmt = null;
    ResultSet rs = null;
    
    try {
        con = getConnection();
        pstmt = con.prepareStatement(sql);
        pstmt.setString(1, userid);
        rs = pstmt.executeQuery();
        
        if(rs.next()) {
            // 🆕 NPE 방지: 객체 생성 후 값 설정
            mvo = new MemberVO();
            mvo.setName(rs.getString("name"));
            mvo.setUserid(rs.getString("userid"));
            mvo.setPwd(rs.getString("pwd"));
            mvo.setEmail(rs.getString("email"));
            mvo.setPhone(rs.getString("phone"));
            mvo.setAdmin(rs.getInt("admin"));
        }
    } catch(Exception e) {
        e.printStackTrace();
    } finally {
        // 리소스 정리
        try {
            if(rs != null) rs.close();
            if(pstmt != null) pstmt.close();
            if(con != null) con.close();
        } catch(Exception e) {
            e.printStackTrace();
        }
    }
    
    return mvo;
}
```

**🔧 개선 사항:**
- `mvo = new MemberVO()` 추가로 NPE 방지
- 완전한 회원 정보 조회 및 반환

## 3. 로그인 프로세스 상세 설명

### 🔄 전체 흐름도 (업데이트됨)

```
[1단계] 사용자가 브라우저에서 /login.do 접속
   ↓
[2단계] LoginServlet의 doGet() 실행
   ↓
[3단계] member/login.jsp 화면 표시 (세련된 CSS 적용)
   ↓
[4단계] 사용자가 아이디/비밀번호 입력 후 "로그인" 버튼 클릭
   ↓
[5단계] JavaScript 검증 (loginCheck 함수)
   ├─ 아이디 비어있음 → "아이디를 입력해주세요" 팝업
   ├─ 비밀번호 비어있음 → "비밀번호를 입력해주세요" 팝업
   └─ 모두 입력됨 → 다음 단계로 진행
   ↓
[6단계] 폼 데이터가 LoginServlet의 doPost()로 전송 (절대 경로)
   ↓
[7단계] LoginServlet이 아이디/비밀번호 받기
   ↓
[8단계] MemberDAO.userCheck() 호출
   ↓
[9단계] MemberDAO가 DB에서 확인
   ├─ DB 연결 (getConnection)
   ├─ SQL 쿼리 실행 ("select pwd from member where userid = ?")
   └─ 결과 확인
   ↓
[10단계] 결과 반환
   ├─ result = 1 → 로그인 성공
   ├─ result = 0 → 로그인 실패
   └─ result = -1 → DB 오류
   ↓
[11단계] 🆕 LoginServlet이 결과에 따라 처리
   ├─ 성공: 
   │   ├─ 세션에 userid 저장
   │   ├─ 🆕 MemberDAO.getMember()로 완전한 회원 정보 조회
   │   ├─ 🆕 세션에 loginUser(MemberVO) 저장
   │   └─ 🆕 /member/main.jsp로 포워딩
   └─ 실패: 
       ├─ 🆕 에러 메시지 설정
       ├─ 🆕 아이디 값 유지 (request.setAttribute)
       └─ 🆕 /member/login.jsp로 포워딩
   ↓
[12단계] 🆕 최종 화면 표시
   ├─ 성공: main.jsp (세련된 메인 화면)
   └─ 실패: login.jsp (아이디 값 유지된 로그인 화면)
```

## 4. 코드 한 줄씩 이해하기

### 🔍 🆕 main.jsp 핵심 코드

#### 1. JSTL 로그인 체크

```jsp
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>

<c:if test="${empty loginUser}">
    <jsp:forward page="/login.do" />
</c:if>
```

**분해 설명:**
- `<%@ taglib %>`: JSTL 라이브러리 사용 선언
- `<c:if test="${empty loginUser}">`: 세션에 `loginUser`가 없으면
- `<jsp:forward page="/login.do" />`: 로그인 페이지로 자동 이동
- **보안 효과**: 로그인하지 않은 사용자는 메인 페이지 접근 불가

#### 2. EL 표현식으로 데이터 표시

```jsp
<span class="welcome-user">${loginUser.name}</span>님
<p>ID : <strong>${loginUser.userid}</strong></p>
<p>Admin : <strong>${loginUser.admin == 1 ? '관리자' : '일반회원'}</strong></p>
```

**분해 설명:**
- `${loginUser.name}`: 세션의 `loginUser` 객체에서 `name` 속성 가져오기
- `${loginUser.admin == 1 ? '관리자' : '일반회원'}`: 삼항 연산자 사용
- **장점**: Java 코드 없이 깔끔한 데이터 표시

## 5. 문제 해결 과정 정리

- 로그인 성공 후 예전 HTML이 표시됨
- 새로 만든 `main.jsp`로 이동하지 않음

**원인:**
```java
// 기존 문제 코드
url = "main.jsp";  // 상대 경로 → 루트의 main.jsp 참조
```

**해결:**
```java
// 수정된 코드
url = "/member/main.jsp";  // 절대 경로 → 올바른 main.jsp 참조
```

**교훈**: 절대 경로 사용으로 경로 혼동 방지

#### 문제 2: 🆕 MemberDAO NPE 발생

**증상:**
- `dao.getMember()` 호출 시 NullPointerException 발생
- 로그인 성공 후 세션에 null 저장됨

**원인:**
```java
// 기존 문제 코드
if(rs.next()) {
    mvo.setName(rs.getString("name"));  // mvo가 null인 상태!
}
```

**해결:**
```java
// 수정된 코드
if(rs.next()) {
    mvo = new MemberVO();  // 🆕 객체 생성 후
    mvo.setName(rs.getString("name"));  // 값 설정
}
```

**교훈**: 객체 사용 전 반드시 생성 확인

#### 문제 3: 🆕 로그인 실패 시 아이디 사라짐

**증상:**
- 로그인 실패 후 아이디 입력 필드가 비워짐
- 사용자가 아이디를 다시 입력해야 함

**원인:**
- 실패 시 입력한 아이디 값을 전달하지 않음

**해결:**
```java
// LoginServlet에서
request.setAttribute("userid", userid);
```

```jsp
<!-- login.jsp에서 -->
<input type="text" name="userid" value="${empty userid ? '' : userid}">
```

**교훈**: 사용자 경험을 위한 데이터 유지 중요

---

#### 문제 4: 🆕 CSS 스타일 적용 안됨

**증상:**
- CSS 파일을 만들었지만 스타일이 적용되지 않음
- 브라우저에서 404 에러 발생

**원인:**
- 상대 경로로 CSS 파일 참조
- 서버 재시작 없이 정적 리소스 변경

**해결:**
```jsp
<!-- 절대 경로 사용 -->
<link rel="stylesheet" type="text/css" 
      href="<%=request.getContextPath()%>/css/member-main.css">
```

**추가 조치:**
- 서버 재시작
- 브라우저 캐시 클리어 (Ctrl+F5)
