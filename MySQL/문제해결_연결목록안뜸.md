# 연결 목록이 안 뜰 때 해결 방법

## 🔍 체크리스트

### 1. 확장 프로그램 설치 확인
- Database Client 확장 프로그램이 설치되어 있는지 확인
- 확장 프로그램이 활성화되어 있는지 확인
- 확장 프로그램 아이콘이 왼쪽 사이드바에 표시되는지 확인

### 2. 설정 파일 형식 확인
Database Client 확장 프로그램은 다음과 같은 형식을 사용합니다:

**올바른 형식:**
```json
[
  {
    "name": "연결이름",
    "host": "localhost",
    "port": 3306,
    "username": "root",  // ⚠️ "user"가 아닌 "username" 사용
    "password": "비밀번호",
    "database": "데이터베이스명",
    "driver": "MySQL",
    "connectionType": "direct",
    "ssl": false,
    "group": "Local"
  }
]
```

**주요 필드:**
- `name`: 연결 이름 (필수)
- `username`: 사용자명 (⚠️ "user"가 아님)
- `password`: 비밀번호
- `host`, `port`, `database`: 연결 정보
- `driver`: "MySQL" (필수)
- `ssl`: true/false

### 3. 설정 파일 위치 확인
확장 프로그램 설정 파일 위치:
```
~/Library/Application Support/Cursor/User/globalStorage/cweijan.vscode-database-client2/connections.json
```

### 4. Cursor 재시작
설정 파일을 수정한 후에는 **반드시 Cursor를 완전히 종료하고 재시작**해야 합니다.

### 5. 수동으로 연결 추가하기
GUI를 통해 수동으로 연결을 추가해보세요:
1. 왼쪽 사이드바에서 Database Client 아이콘 클릭
2. "+" 버튼 또는 "Add Connection" 클릭
3. 연결 정보 입력:
   - Name: Local instance 3306
   - Host: localhost
   - Port: 3306
   - Username: root
   - Password: [비밀번호]
   - Database: dcproject
   - SSL: 끄기

### 6. 다른 확장 프로그램 사용
Database Client가 작동하지 않는다면, 다른 MySQL 확장 프로그램을 사용할 수 있습니다:

- **SQLTools + MySQL Driver**: `.sqltoolsrc.json` 파일 사용
- **MySQL (cweijan)**: 다른 설정 형식 사용 가능

## 🔧 시도해볼 방법

### 방법 1: 설정 파일 형식 변경
현재 파일을 `username` 형식으로 변경했습니다. Cursor를 재시작해보세요.

### 방법 2: 확장 프로그램 재설치
1. 확장 프로그램 제거
2. Cursor 재시작
3. 확장 프로그램 재설치
4. GUI로 연결 추가

### 방법 3: 다른 설정 파일 위치 확인
일부 확장 프로그램은 다른 위치에 설정을 저장할 수 있습니다:
- `~/.vscode/database.json`
- `~/.vscode/database-client.json`
- 프로젝트 폴더의 `.vscode/settings.json`
