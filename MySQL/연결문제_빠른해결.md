# MySQL 연결 문제 빠른 해결 가이드

## 현재 상황
- ✅ MySQL 서버 실행 중
- ✅ 포트 3306 정상 작동
- ❌ root 계정 비밀번호 필요

## 빠른 해결 방법

### 방법 1: 비밀번호 확인 후 설정 파일 업데이트 (권장)

1. **Workbench에서 비밀번호 확인**
   - MySQL Workbench를 열고 연결 설정 확인
   - 사용 중인 root 비밀번호 확인

2. **설정 파일 업데이트**
   ```bash
   # 비밀번호를 알고 있다면
   ./update_connection_password.sh [비밀번호]
   
   # 예: 비밀번호가 password1234인 경우
   ./update_connection_password.sh password1234
   ```

3. **Cursor 재시작**
   - Cursor를 완전히 종료하고 재시작
   - Database Client에서 연결 테스트

### 방법 2: 비밀번호 재설정

비밀번호를 모르거나 잊었다면:

```bash
# 비밀번호 재설정 스크립트 실행
./reset_mysql_password_final.sh
```

스크립트가 자동으로:
- MySQL 안전 모드로 시작
- 새 비밀번호 설정
- 정상 모드로 재시작
- 연결 설정 파일 업데이트

### 방법 3: 수동 연결 테스트

```bash
# 연결 테스트
./test_mysql_connection.sh

# 또는 직접 연결
mysql -u root -p
```

## 단계별 해결 체크리스트

1. [ ] 비밀번호 확인 (Workbench 또는 기억)
2. [ ] `./update_connection_password.sh [비밀번호]` 실행
3. [ ] Cursor 재시작
4. [ ] Database Client에서 연결 테스트
5. [ ] 연결 실패 시 `./reset_mysql_password_final.sh` 실행

## 문제가 계속되면

### 추가 진단
```bash
# 연결 테스트
./test_mysql_connection.sh

# MySQL 서버 상태 확인
brew services list | grep mysql

# 포트 확인
lsof -i :3306
```

### 다른 사용자로 접속 시도
root 대신 jdbctest 사용자로 접속:
- Username: `jdbctest`
- Password: jdbctest 비밀번호

## 설정 파일 위치

연결 설정이 저장된 파일들:
- `config.json`
- `.vscode/database-connections.json`
- `~/Library/Application Support/Cursor/User/globalStorage/cweijan.vscode-database-client2/connections.json`

모든 파일의 비밀번호가 일치해야 합니다.
