# MySQL 비밀번호 오류 해결 방법

## 🔴 오류 메시지
```
Access denied for user 'root'@'localhost' (using password: YES)
```

이 오류는 **비밀번호가 잘못되었거나** root 계정 접속 권한이 없을 때 발생합니다.

---

## 🔧 해결 방법

### 방법 1: 비밀번호 확인 및 재입력

1. **Workbench에서 사용하는 비밀번호 확인**
   - Workbench에서 성공적으로 연결된 비밀번호를 사용
   - Database Client 연결 설정에서 비밀번호가 정확한지 확인

2. **비밀번호 재입력**
   - Database Client 연결 설정에서 비밀번호 필드를 비우고 다시 입력
   - 저장 후 재연결 시도

---

### 방법 2: 다른 사용자로 접속 시도

root 대신 **jdbctest** 사용자로 접속해보세요:

1. Database Client에서 **jdbctest** 연결 사용
2. 또는 새 연결 생성:
   - Username: `jdbctest`
   - Password: jdbctest 비밀번호

---

### 방법 3: MySQL 비밀번호 재설정 (root)

터미널에서 MySQL을 안전 모드로 실행하여 비밀번호를 재설정:

#### macOS에서:

```bash
# 1. MySQL 서비스 중지 (만약 실행 중이라면)
sudo /usr/local/mysql/support-files/mysql.server stop

# 2. MySQL 안전 모드로 시작 (비밀번호 없이 접속)
sudo mysqld_safe --skip-grant-tables &

# 3. 새 터미널에서 MySQL 접속 (비밀번호 없이)
mysql -u root

# 4. MySQL 안에서 비밀번호 재설정
USE mysql;
ALTER USER 'root'@'localhost' IDENTIFIED BY '새비밀번호';
FLUSH PRIVILEGES;
EXIT;

# 5. MySQL 서비스 재시작
sudo /usr/local/mysql/support-files/mysql.server restart
```

#### Homebrew로 설치한 경우:

```bash
# MySQL 서비스 중지
brew services stop mysql

# 안전 모드로 시작
mysqld_safe --skip-grant-tables &

# 새 터미널에서 접속
mysql -u root

# 비밀번호 재설정
USE mysql;
ALTER USER 'root'@'localhost' IDENTIFIED BY '새비밀번호';
FLUSH PRIVILEGES;
EXIT;

# 서비스 재시작
brew services start mysql
```

---

### 방법 4: 비밀번호 없이 접속 시도 (테스트용)

**⚠️ 보안상 권장하지 않지만, 테스트용으로:**

```bash
# 비밀번호 없이 접속 시도
mysql -u root -h localhost

# 또는
mysql -u root -h 127.0.0.1
```

만약 이것도 안 되면 root 계정이 비밀번호를 요구하도록 설정되어 있는 것입니다.

---

### 방법 5: Workbench에서 비밀번호 확인

Workbench에서 연결 설정을 확인:

1. Workbench 열기
2. 연결 설정 편집
3. 비밀번호 필드 확인 (보안상 ***로 표시되지만)
4. 같은 비밀번호를 Database Client에 입력

---

## 🎯 권장 해결 순서

1. ✅ **Workbench에서 사용하는 비밀번호 재확인**
2. ✅ **Database Client 연결 설정에서 비밀번호 재입력**
3. ✅ **jdbctest 사용자로 접속 시도** (root 대신)
4. ✅ **MySQL 비밀번호 재설정** (위 방법 3 참고)

---

## 💡 팁

### jdbctest 사용자로 데이터베이스 생성

root 접속이 안 되면, jdbctest 사용자로 접속하여 데이터베이스를 생성할 수 있습니다:

```sql
-- jdbctest로 접속 후 (권한이 있다면)
CREATE DATABASE IF NOT EXISTS `dcproject`
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;
```

만약 jdbctest에 CREATE DATABASE 권한이 없다면, root로 접속해야 합니다.

---

## ⚠️ 주의사항

- 비밀번호 재설정 시에는 MySQL 서비스가 중단될 수 있습니다
- 프로덕션 환경에서는 주의해서 진행하세요
- 비밀번호를 변경한 후에는 모든 연결 설정에서 비밀번호를 업데이트해야 합니다
